EDT Keypad navigation is from a time before keyboards had arrow keys let alone PageUp/Down keys. DEC software engineers made efficient use of the numeric keypad that was added for data entry users. Instead of a big plus key there was a minus and comma key. The top 4 keys were not math operators; rather they were PF-1 through PF-4. In order they were GOLD, HELP, Find, Delete Line.

Unlike a PC where you have to hold down and another key, you simply press and release the GOLD key then press and release the key for the function you want. The text that doesnâ€™t have a gold background is what they key does when the GOLD key was not pressed in front of it.

Because some EDT emulation modes on some editors, notably Emacs, are unable to map the NumLock key as GOLD, the Option dialog allows you to choose between NumLock and ScrollLock as your GOLD key.

EDT mode is direction maintaining. The default direction is Advance [the 4 key]. You can change the direction to Backup by hitting the 5 key. All of your navigation operations will continue in that direction until you change direction again.

All you really need to know is HELP

The PF2 key [/ on the PC keypad] brings up this Help text. Many a programmer learned EDT without access to a printed manual and before the Internet was even a dream. The Help screen had the keypad image you see here and just a few lines of text explaining a few cryptic things. We didn't have screen capture utilities on those green serial terminals, but some of them had an extra serial port for a printer. Many a developer used the magic terminal key combination to print the Help screen on an attached printer then hung it in their cubicle. This Help screen has far more information than we ever had.

NOTE: Because EDT was from a time before desktop computers or the concept of desktop software, there was no OS or desktop level clipboard. EDT was a mult-buffer editor ahead of its time. Today you have multiple tabs and think nothing of it. One buffer that was always created was PASTE. When you selected and cut text it went here. You could append to it, view it, and even edit it. This EDT emulation will use the system supplied clipboard. It will let you append and view the clipboard contents.

EDT became a favorite editor for tens of thousands of developers. It wasn't "expert friendly" like most of the editors then and now. All you needed was that screen print and someone telling you how to exit and save. All of the most used things were on the keypad. There were only a token few commands to execute at the command prompt and you didn't need them very often. You only had to know one: EXIT

In general, just be aware that most of the "delete" functions can be undone using GOLD and the same key. The exception is DEL EOL which is discussed later.


Navigation

LINE [the 0 key] moves a direction sensitive line at a time.
WORD [the 1 key] moves a direction sensitive word (determined by whitespace) at a time
EOL [the 2 key] moves to the directional end of line. In Backup mode this is the beginning
CHAR [the 3 key] moves a direction sensitive character at a time
ADVANCE [the 4 key] Sets the current editing direction forward which is the default at startup
BOTTOM [GOLD ADVANCE] Takes you to end of file no matter what direction you have set.
BACKUP [the 5 key] Sets the current editing direction backwards towards the beginning of the file
TOP [GOLD BACKUP] Takes you to top of file no matter what direction you have set.
PAGE [the 7 key] actually looks for a FormFeed character. SPECINS (short for special insert) is how you actually enter ASCII character numeric values. Developers printed on continuous form back then and we had to provide the page breaks in the source. This is still really handy when working with languages or data that has defined sections. By placing a FormFeed at the start of each region you can jump between them.
SECT [the 8 key] is what most closely equates to PC Page Up/Down. It moves 16 lines at a time.

Editing

DEL L [PF4 - the minus key on the PC keypad] (Delete Line) Deletes from cursor THROUGH directional end of line removing the end of line character causing the text below to move up.
DEL EOL [GOLD 2 key] Deletes from where your cursor is to the directional End of Line. DOES NOT REMOVE THE EOL character(s)
UND L [GOLD PF4 - the minus key on the PC keypad] As weird as it may sound, both DEL L and DEL EOL used the same cut buffer. Using this UnDelete will paste the contents of whichever was last used where your cursor is currently located.
DEL W [default map + key on the PC keypad] The option screen lets you chose a different GOLD key mapping because this is the missing keypad key. Some developers use Delete Word all of the time and others (like myself) never do. Keep in mind that word deletion like most things honors the current editing direction
UND W [GOLD DEL W] Pastes the last deleted word where your cursor is in a directionally sensitive manner.
DEL C [the + key on PC keypad] Deletes the character under your cursor and moves in the current editing direction.
SELECT[the . key on the PC keypad] Begins a selection area to be part of another operation.
RESET [GOLD SELECT] Cancels current selection area.
SPECINS [GOLD CHAR] Prompts for a numeric character value which then gets inserted where the cursor is in the file. May support x and u prefixed values for Hex and Unicode. At least that is a planned enhancement. The most used value historically was 12 for ASCII FormFeed because that was used by the PAGE key to identify the start of another page. Some scripts had need of ASCII 7 (Bell) and others needed ASCII 9 (hard horizontal tab).
CUT [the 6 key on the PC keypad] Once you have SELECTed an area you can CUT it into the paste buffer. On PCs this is typically also called the clipboard.
PASTE [GOLD CUT] Pastes the current contents of the PASTE buffer where your cursor is.
APPEND [the 9 key on the PC keypad] Deletes the selected range of text and appends it to the end of the paste buffer (clipboard)
REPLACE [GOLD APPEND] Deletes the currently selected text and replaces it with the text from the paste buffer (clipboard)
CHNGCASE [GOLD WORD (the 1 key)] If you have no current selection this will change the case of the character "under" the cursor. This editor places the cursor "between" characters so the case will be changed for the directionally sensitive "next" character. When you have a current selection the case of every letter is changed. Not implemented

COMMAND There are a small number of EDT commands that would still prove useful in this day and age, but none have been implemented.

SUBS By today's standard this was a complex key sequence for making a text substitution then finding the next matching text. I spent 20 years on DEC platforms and never used it. The SUBS command was much easier and less prone to user error. When you hit the SUBS key combination you will be presented with the "Find" dialog

FILL In the very old days when we never turned on wordwrap for fear of forgetting it and trashing a large source file, FILL was necessary. You selected a bunch of lines and it would "pack" as many words as possible without splitting a word on each line. When we only had an 80 column screen and narrow continuous form only allowed for 80 printed columns this was how we made our documentation look like we cared. Today you have wordwrap and many free word processors. This editor has wordwrap and spell check. It also has a rewrap option on the context menu. It just wasn't worth the effort to implement FILL.

Some Apple/MAC users will be a bit hosed

Most of you probably won't be impacted. If you have on of the Apple/MAC keyboards that sends the numeric keypad modifier with every arrow key then you won't be able to hit the EDT SELECT key and use the arrow keys to make your selection. This could also impact certain laptop users where the keyboard doesn't have an independent numeric keypad. EDT Navigation is meant to be used with a numeric keypad. If your OS or keyboard had to do something cute to simulate such a thing you probably shouldn't turn on the support. You can still navigate by character, line, and word; you just have to use the EDT keys.
